# Dark Mode Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add full dark mode support with system-default + manual override, using a ThemeContext that provides typed color tokens to all 37+ components.

**Architecture:** A new `lib/theme.ts` defines `lightColors`/`darkColors` token objects. `lib/ThemeContext.tsx` merges `useColorScheme()` with a persisted `themeOverride` from `stores/themeStore.ts`. Every component replaces its module-level `StyleSheet.create({...hardcoded...})` with a `createStyles(c: Colors)` factory called inside `useMemo`.

**Tech Stack:** React Native, Expo, Zustand + `zustand/middleware` persist, `@react-native-async-storage/async-storage`

```typescript
// stores/themeStore.ts
import AsyncStorage from '@react-native-async-storage/async-storage'
import { create } from 'zustand'
import { persist, createJSONStorage } from 'zustand/middleware'

export type ThemeOverride = 'light' | 'dark' | 'system'

type ThemeStore = {
  themeOverride: ThemeOverride
  setThemeOverride: (v: ThemeOverride) => void
}

export const useThemeStore = create<ThemeStore>()(
  persist(
    (set) => ({
      themeOverride: 'system',
      setThemeOverride: (v) => set({ themeOverride: v }),
    }),
    {
      name: 'lso-theme',
      storage: createJSONStorage(() => AsyncStorage),
    }
  )
)
```

**Files:**

- [ ] **Step 1: Create context and hook**

```typescript
// lib/ThemeContext.tsx
import React, { createContext, useContext, useMemo } from 'react'
import { useColorScheme } from 'react-native'
import { lightColors, darkColors, Colors } from './theme'
import { useThemeStore } from '../stores/themeStore'

type ThemeContextValue = {
  colors: Colors
  isDark: boolean
  scheme: 'light' | 'dark'
}

const ThemeContext = createContext<ThemeContextValue>({
  colors: lightColors,
  isDark: false,
  scheme: 'light',
})

export function ThemeProvider({ children }: { children: React.ReactNode }) {
  const systemScheme = useColorScheme() ?? 'light'
  const themeOverride = useThemeStore((s) => s.themeOverride)

  const scheme: 'light' | 'dark' =
    themeOverride === 'system' ? systemScheme : themeOverride

  const isDark = scheme === 'dark'
  const colors = isDark ? darkColors : lightColors

  const value = useMemo(
    () => ({ colors, isDark, scheme }),
    [colors, isDark, scheme]
  )

  return (
    <ThemeContext.Provider value={value}>
      {children}
    </ThemeContext.Provider>
  )
}

export function useTheme(): ThemeContextValue {
  return useContext(ThemeContext)
}
```

## Task 6: Systematic component refactor — color token mapping

**Files:** All 37 `.tsx` files listed at end of this task

This task applies the SAME mechanical pattern to all component files. Work through each file in the list below.

### Pattern to apply to EVERY file

**1. Add imports** (at top, after existing imports):
```tsx
import { useTheme } from '../../lib/ThemeContext'   // adjust depth: ../../ or ../../../
import { useMemo } from 'react'                       // add to existing react import if missing
import { Colors } from '../../lib/theme'              // adjust depth
```

**2. Add inside component** (before `return`):
```tsx
const { colors: c } = useTheme()
const styles = useMemo(() => createStyles(c), [c])
```

**3. Move StyleSheet outside component** and convert to function:
```tsx
// BEFORE (at module level):
const styles = StyleSheet.create({ ... })

// AFTER (outside component, after component function):
function createStyles(c: Colors) {
  return StyleSheet.create({ ... })
}
```

**4. Replace inline color props in JSX:**
```tsx
// BEFORE:
<Ionicons color="#1A237E" />
<Ionicons color="#0D1B2A" />

// AFTER:
<Ionicons color={c.primary} />
<Ionicons color={c.text} />
```
